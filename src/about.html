Arbetsprocessen

Som verktyg för att skapa den här webbplatsen använde jag en dator med windows 11 där programmen Visual Studio Code, Git och NodeJS med pakethanteraren NPM är installerat. För att remote versionshantera och för hjälp vid publisering användes GitHub och den är publiserad på Netlify.

Jag började med att initiera ett nytt NPM projekt. För att göra det navigerade jag till min tänkta projektmapp i kommandotolken och skrev kommandot "npm init". Då skapades en konfigurationsfil med namnet package.json. Sen skrev jag "npm install parcel --save-dev" för att installare paketet parcel till mitt projekt för developer-läget. I projektmappen skapade jag en fil med namnet .gitignore för att utesluta mapparna node_modules, dist och parcel-cache för att de inte ska versionshanteras med git. En GitHub remote repositorie anslöts och förgreningen developer skapades och aktiverades i Visual Studio Code. Filen med namnet sharp.config.json skapades för att ställa kvaliten för bilder till 70. Ändringar gjordes också i filen package.json där source bestämdes till index.html och skripten start: parcel och build: parcel build lades till. Jag skapade mappen src för att ha källkod och alla orginalfiler i och i den skapade jag mappar för varje filtyp som skulle vara med till webbplatsen, css, js, pictures och svg.

Webbplatsen skrevs i html, css och JavaScript med hjälp av Visual Studio Code. JavaScript används enbart till mobilmenyn. Jag fotograferade fem foton på min kaffehörna. I html-koden till bildersidan skapade jag en picture-tagg för varje bild. Här användes srcset och img-taggen för att både kunna använda modern och äldre standard för bilder. Eftersom att jag ville visa bilden i 600px bredd med formatet webp skrev jag "source srcset="pictures/filter.jpeg?as=webp&width=600" type="image/webp"". Då konverteras och komprimeras bilden automatiskt när paketet parcel körs. För min backup använde jag koden "img src="pictures/filter.jpeg?width=600" width="600"".

När webbplatsen var klar merchade jag mina Git-förgreningar så att projecten även hamnade i main förgreningen som skulle publiseras. Eftersom att mitt GitHub-konto redan var kopplat till Netlify var det bara att leta reda på projektet på Netlifys webbplats. Jag valde projektet och skrev in byggkommandot "npm run build" och tryckte deploy.

De här optimeringarna gjorde parcel vid publicering

  • Minifiera CSS-koden
  • Transpilera och minifiera JavaScript-koden
  • Minifiera HTML-koden
  • Komprimering och bildoptimering till olika format
  • JavaScript-koden Transpileras för kompabilitet
  • Optimerade arbetsfiler kopieras till publiceringsmapp
